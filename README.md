# 前言：为什么要用线性代数的视角理解 LLM

## 一个程序员的自白

2023年的某个深夜，我正对着屏幕上的ChatGPT发呆。

我输入了一段话，它流畅地回复了一篇像模像样的产品方案——逻辑清晰、措辞得体，甚至主动补充了我没想到的风险点。我忍不住想：这东西到底是怎么做到的？

我不是那种看到新技术就只会喊"魔法"的人。我写了十多年代码，对"黑箱"有一种本能的不甘。于是我开始翻论文——先是 Attention Is All You Need，然后是 GPT 系列的技术报告，再然后是各种开源模型的架构文档。

然后我就撞墙了。

满纸的矩阵乘法、线性变换、特征值分解、低秩近似……每一个词我都见过，但没有一个是真正理解的。大学线性代数课上学的那点东西——如果你跟我一样，大概就是背公式、算行列式、考完就忘——在这片新大陆上完全不够用。

那一刻我意识到：**线性代数不是理解 LLM 的加分项，它是唯一的入场券。**

## 为什么是线性代数

大语言模型的全部计算——从输入到输出——本质上就是一系列矩阵乘法和向量运算。词被表示成向量，权重是矩阵，注意力是矩阵乘以矩阵的转置再乘矩阵，Softmax 作用在向量上，位置编码是向量的旋转，LoRA 的核心思想是低秩分解……

**这里没有比喻，没有夸张。这就是字面意义上的事实。**

当你把一个 prompt 输入 GPT，它在内部做的事情可以这样概括：你的文字被拆成 token，每个 token 变成一个高维向量（比如 4096 维），然后这些向量经过几十层线性变换——每一层都是权重矩阵乘以输入向量，加上偏置，再经过激活函数。几十层之后，最后一个向量被映射回词表空间，变成下一个 token 的概率分布。

整个过程，从头到尾，就是线性代数。

你可以不懂线性代数而"使用"大模型——就像你可以不懂发动机原理而开车一样。但如果你想在以下任何一个方向走得更远，线性代数就是不可回避的：

- **理解模型行为**：为什么它会产生幻觉？为什么 attention 会关注某些 token？
- **训练和微调**：LoRA 为什么能以极少参数高效微调？梯度如何流动？
- **推理优化**：KV Cache 为什么能加速推理？模型量化的数学含义是什么？
- **架构创新**：想要提出新的 attention 机制、新的位置编码方案，你需要能读懂论文里的每一个公式。

## 这本书有什么不一样

市面上线性代数的书汗牛充栋，深度学习的书也越来越多。但它们之间有一条巨大的裂缝——线性代数教材从定义出发，学完一整本书不知道跟真实世界有什么关系；深度学习书则默认你已有数学基础，公式一个接一个，从来不解释"为什么"。

**这本书要做的是在这条裂缝上架一座桥。**

每一个概念都遵循四步结构：

**第一步，生活类比。** 我们不会上来就甩定义。向量不只是"有序的数字列表"——它是你在地图上的位置，是语义空间中一个词的"坐标"。矩阵不只是"数字排成矩形"——它是自动化加工流水线，是把一个空间"揉捏"成另一个空间的操作手册。

**第二步，数学概念。** 从类比过渡到精确的数学表述。这里会有公式，但每一个公式出现时，你都已经知道它"在说什么"：

$$A \mathbf{x} = \mathbf{b}$$

这个等式在说：把向量 $\mathbf{x}$ 经过矩阵 $A$ 描述的变换，得到向量 $\mathbf{b}$。就这么简单。

**第三步，连接到 LLM。** 每个数学概念直接对应到大语言模型中的真实机制——注意力头中的 $QK^T$ 运算、前馈网络中的矩阵乘法、位置编码中的旋转变换——用刚学的数学去拆解它们。

**第四步，Python 演示。** 每个概念都配有可直接运行的 Python 代码（主要使用 NumPy 和 PyTorch），让你亲手验证刚才学到的数学。看到矩阵运算的结果从代码中输出，那种"原来如此"的感觉比读十遍定义都管用。

全书的 14 章分为四个部分：

- **第一部分"旧友重逢"**（第 1-3 章）：向量、矩阵、线性变换——重拾线性代数记忆
- **第二部分"核心武器"**（第 4-7 章）：秩、分解、特征值、SVD——理解现代 AI 的关键钥匙
- **第三部分"拆解 Transformer"**（第 8-11 章）：张量、注意力、位置编码、Softmax——逐层拆开 Transformer
- **第四部分"进阶前沿"**（第 12-14 章）：LoRA、梯度流动、KV Cache——走向前沿

## 给你的阅读建议

**不要跳过类比。** 如果你是有经验的程序员，可能会觉得类比"太简单"。请不要这样做。这些类比是后续理解数学含义的直觉锚点。线性代数的公式本身并不难，难的是理解它"在说什么"——而类比正是那个帮你开口的工具。

**动手跑代码。** 每章的 Python 代码都很短，但请你真的打开终端跑一遍，改改参数看看结果怎么变。数学在纸上是一种感觉，在代码里是另一种感觉，两种加在一起才是完整的理解。环境很简单：Python 3.8 以上，装好 NumPy 和 PyTorch 即可。

**容忍"暂时不完全懂"的感觉。** 有些概念在第一次出现时你不会完全理解——比如特征值分解要到第 6 章才深入讲解，但前面可能就会提到。这是正常的，因为线性代数的概念之间是网状关联的。先让它过去，后面的章节会从不同角度再次照亮它。

**关注几何直觉。** 线性代数有两种语言：代数（公式和计算）和几何（变换和空间）。很多教材过度偏重前者，导致学生觉得线代就是算数。这本书会特别强调几何视角——向量的方向和长度、线性变换如何"拉伸和旋转"空间。当你能在脑海里"看到"这些操作，公式就不再抽象了。
