# 第4章 矩阵的深层属性——秩、奇异与信息量

你在公司开会，会议室里坐了十个人。老板抛出一个问题，让大家依次表态。你满怀期待地等着，结果发现：张三说完，李四说"我同意张三"，王五说"我跟张三一样"，赵六说"张三说得好，我也是"。十个人里，真正独立思考并给出了独特观点的，只有三个人。其余七个人，本质上只是把张三的话换了个说法。

这个会议的"秩"就是 3。

秩（Rank）是线性代数中最深刻、也最常被低估的概念。它不只是一个矩阵的数值属性——它告诉你这个矩阵里到底装了多少"真正的信息"。理解了秩，你就理解了为什么有些矩阵能完美反转回去（可逆），有些矩阵一操作就丢信息（奇异），以及为什么在 LLM 的世界里，低秩近似是一个如此强大的工具。

---

## 4.1 什么是矩阵的秩

### 从"独立信息量"的角度理解秩

回到开会的类比。假设你们有五个同事对一个技术方案发表意见：

- **小王**说："我建议用微服务架构。"
- **小李**说："我同意小王，微服务好。"
- **小张**说："我也同意小王。"
- **小赵**说："我认为应该用单体架构。"
- **小刘**说："我支持小赵的看法。"

五个人的发言里，有多少条**独立**的观点？两条：小王的"微服务"和小赵的"单体"。小李和小刘只是在复读，小张也是在跟风。所以这个讨论的"秩"是 2。

矩阵也是一样的道理。一个矩阵有很多列向量（就好比很多个人的发言），但有些列只是其他列的"复读机"。秩告诉你：**这些列里，到底有多少个是在说不同的事情。**

### 秩的数学定义

**矩阵的秩（rank）等于其列向量中线性无关（linearly independent）的最大个数。**

什么是线性无关？一组向量 $\mathbf{v}_1, \mathbf{v}_2, \ldots, \mathbf{v}_k$ 是线性无关的，意味着它们中没有任何一个可以被其余向量的线性组合表示出来。换句话说，不存在不全为零的标量 $c_1, c_2, \ldots, c_k$ 使得：

$$c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \cdots + c_k\mathbf{v}_k = \mathbf{0}$$

如果存在这样的组合，那说明至少有一个向量是"多余的"——它可以被其他向量拼出来，没有带来新的信息。

**线性相关的几何含义：谁在复读。** 那个代数条件 "$c_1\mathbf{v}_1+c_2\mathbf{v}_2+\cdots+c_k\mathbf{v}_k=\mathbf{0}$（系数不全为零）" 到底在说什么？把它翻译成几何：如果存在不全为零的系数能凑出零向量，不妨设 $c_3\ne0$，移项就得到

$$\mathbf{v}_3 = -\frac{c_1}{c_3}\mathbf{v}_1 - \frac{c_2}{c_3}\mathbf{v}_2$$

也就是说，$\mathbf{v}_3$ 能被其余向量拼出来——它**落在其余向量张成的空间里**，没有贡献任何新方向。所以：

- **线性相关** ⟺ 至少有一个向量落在其余向量张成的空间里（是"复读机"）；
- **线性无关** ⟺ 每个向量都贡献一个其余向量凑不出来的全新方向。

一个直接的推论：在 2 维平面里，任意 3 个向量必然线性相关——因为它们都挤在同一个 2 维空间里，第三个迟早落在前两个张成的范围内。所以 2 维空间最多容纳 2 个线性无关的向量，$n$ 维空间最多容纳 $n$ 个。

**一个 2 维小例：** $\mathbf{v}_1=[1,0]$（指向 $x$ 轴）、$\mathbf{v}_2=[0,1]$（指向 $y$ 轴）线性无关，各自贡献一个方向。再加入 $\mathbf{v}_3=[1,1]$，它恰好等于 $\mathbf{v}_1+\mathbf{v}_2$——落在两者张成的平面里，没带来新方向。代数上：$1\cdot\mathbf{v}_1+1\cdot\mathbf{v}_2-1\cdot\mathbf{v}_3=\mathbf{0}$，系数不全为零，故三者线性相关。

```python
import numpy as np
v1, v2, v3 = np.array([1.,0.]), np.array([0.,1.]), np.array([1.,1.])
print("v3 == v1+v2 ?", np.allclose(v3, v1+v2))   # True -> v3 冗余, 三者线性相关
print("v1,v2 是否能凑出 0(系数非全0)?", "否 -> 线性无关")
```

> **一句话锚点：** 线性无关 = 每个向量都贡献一个新方向；线性相关 = 有人在复读别人的话。

举个例子：

$$A = \begin{pmatrix} 1 & 2 & 3 \\ 0 & 1 & 1 \\ 1 & 3 & 4 \end{pmatrix}$$

乍看是三列，但仔细观察第三列 $\begin{pmatrix} 3 \\ 1 \\ 4 \end{pmatrix}$，你会发现它恰好等于第一列加第二列：$\begin{pmatrix} 1 \\ 0 \\ 1 \end{pmatrix} + \begin{pmatrix} 2 \\ 1 \\ 3 \end{pmatrix} = \begin{pmatrix} 3 \\ 1 \\ 4 \end{pmatrix}$。第三列没有带来任何新信息。所以 $\text{rank}(A) = 2$，不是 3。

### 行秩 = 列秩

一个有趣的事实：矩阵的行秩和列秩总是相等的。也就是说，"行中有多少独立信息"和"列中有多少独立信息"是同一个数字。这个数字就是矩阵的秩。

这个结论并不显然，但它意味着你可以从行的角度或列的角度来看秩，结果都一样。对于 $m \times n$ 的矩阵 $A$，秩满足：

$$\text{rank}(A) \leq \min(m, n)$$

当 $\text{rank}(A) = \min(m, n)$ 时，我们说矩阵是**满秩**（full rank）的——它装了尽可能多的独立信息。

### Python 演示：观察不同矩阵的秩

```python
import numpy as np

# 满秩矩阵：三列各不相同，互相独立
A = np.array([[1, 0, 1],
              [0, 1, 1],
              [0, 0, 1]])
print(f"A 的秩: {np.linalg.matrix_rank(A)}")  # 3

# 秩为 2：第三列 = 第一列 + 第二列
B = np.array([[1, 2, 3],
              [0, 1, 1],
              [1, 3, 4]])
print(f"B 的秩: {np.linalg.matrix_rank(B)}")  # 2

# 秩为 1：所有行都是第一行的倍数
C = np.array([[1, 2, 3],
              [2, 4, 6],
              [3, 6, 9]])
print(f"C 的秩: {np.linalg.matrix_rank(C)}")  # 1

# 零矩阵：秩为 0
D = np.zeros((3, 3))
print(f"D 的秩: {np.linalg.matrix_rank(D)}")  # 0
```

运行结果很直觉：满秩的 $A$ 秩为 3，有一列是"复读机"的 $B$ 秩为 2，所有行都重复同一信息的 $C$ 秩为 1，什么都没说的 $D$ 秩为 0。

## 4.2 秩一矩阵：矩阵世界的"原子"

### 最简单的非零矩阵

在化学里，原子是能保持化学性质的最小单位。在线性代数里，**秩一矩阵**扮演着类似的角色——它是矩阵世界中不能再"拆"的最小有意义单元。

什么是秩一矩阵？就是一个秩恰好等于 1 的矩阵。它的特点是：**所有行都互为倍数，所有列也都互为倍数。** 整个矩阵只包含"一个方向"的信息。

数学上，任何秩一矩阵都可以写成两个向量的外积：

$$A = \mathbf{u}\mathbf{v}^T$$

其中 $\mathbf{u}$ 是 $m \times 1$ 的列向量，$\mathbf{v}$ 是 $n \times 1$ 的列向量。展开来看：

$$A = \begin{pmatrix} u_1 \\ u_2 \\ \vdots \\ u_m \end{pmatrix} \begin{pmatrix} v_1 & v_2 & \cdots & v_n \end{pmatrix} = \begin{pmatrix} u_1v_1 & u_1v_2 & \cdots & u_1v_n \\ u_2v_1 & u_2v_2 & \cdots & u_2v_n \\ \vdots & \vdots & \ddots & \vdots \\ u_mv_1 & u_mv_2 & \cdots & u_mv_n \end{pmatrix}$$

你看，每一行都是 $\mathbf{v}^T$ 的倍数（倍数为 $u_i$），每一列都是 $\mathbf{u}$ 的倍数（倍数为 $v_j$）。整块矩阵只需要 $m + n$ 个数字就能完全描述（$\mathbf{u}$ 的 $m$ 个分量 + $\mathbf{v}$ 的 $n$ 个分量），而不是 $m \times n$ 个。这就是秩一矩阵的"经济"之处。

### 彩色图像的类比

想象你在看一张彩色照片。这张照片是由三层单色光叠加而成的：红色层、绿色层、蓝色层。每一层本身是一个灰度图（只有一个通道的强度值），但如果把红色层想象成一个只包含"红色方向"信息的矩阵，它就像一个秩一结构——所有像素的色调都指向同一个颜色方向，只是亮度不同。

当然，实际的图像远比秩一矩阵复杂。但这里的关键直觉是：**任何复杂的东西，都可以拆成多层简单东西的叠加。**

### 任意矩阵 = 秩一矩阵的和

这就引出了线性代数中最优美的结论之一：**任意一个秩为 $r$ 的矩阵，都可以写成 $r$ 个秩一矩阵的和。**

这就是**奇异值分解（SVD）**的核心思想：

$$A = \sum_{i=1}^{r} \sigma_i \mathbf{u}_i \mathbf{v}_i^T = \sigma_1\mathbf{u}_1\mathbf{v}_1^T + \sigma_2\mathbf{u}_2\mathbf{v}_2^T + \cdots + \sigma_r\mathbf{u}_r\mathbf{v}_r^T$$

其中 $\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_r > 0$ 是奇异值，$\mathbf{u}_i$ 和 $\mathbf{v}_i$ 分别是左、右奇异向量。

用图像的类比：一张复杂的高分辨率照片，等于很多"单色层"的叠加。每一层就是一个秩一矩阵 $\sigma_i\mathbf{u}_i\mathbf{v}_i^T$，奇异值 $\sigma_i$ 是这层的"亮度"或"重要性"。

**这个分解对 LLM 至关重要。** 后面我们会看到，LoRA（Low-Rank Adaptation）微调技术的全部基础，就建立在这个观察之上：如果一个大权重矩阵的有效秩很低，我们就不需要调整它的全部参数，只需要调整几个秩一矩阵就够了。

### Python 演示：拆解矩阵为秩一矩阵之和

```python
import numpy as np

# 构造一个秩为 2 的 3x3 矩阵
A = np.array([[3, 1, 1],
              [1, 3, 1],
              [1, 1, 3]])

print(f"A 的秩: {np.linalg.matrix_rank(A)}")  # 3... 让我们构造一个真正的秩2矩阵

# 两个秩一矩阵相加，秩最多为 2
u1, v1 = np.array([1, 0, 1]), np.array([1, 2, 0])
u2, v2 = np.array([0, 1, 1]), np.array([1, 0, 2])

R1 = np.outer(u1, v1)  # 秩一矩阵 1
R2 = np.outer(u2, v2)  # 秩一矩阵 2
A = R1 + R2             # 它们的和

print(f"R1 的秩: {np.linalg.matrix_rank(R1)}")  # 1
print(f"R2 的秩: {np.linalg.matrix_rank(R2)}")  # 1
print(f"A = R1 + R2 的秩: {np.linalg.matrix_rank(A)}")  # 2

# 用 SVD 拆解 A，验证可以还原
U, S, Vt = np.linalg.svd(A)
print(f"\n奇异值: {S}")

# 只保留前 2 个奇异值（秩 = 2），重建矩阵
A_reconstructed = U[:, :2] @ np.diag(S[:2]) @ Vt[:2, :]
print(f"\n重建误差: {np.linalg.norm(A - A_reconstructed):.2e}")  # 接近 0
```

你会看到，一个秩为 2 的矩阵，恰好被 2 个秩一矩阵（SVD 的两个分量）完美重建。多一个分量是浪费，少一个分量则丢失信息。

## 4.3 奇异矩阵与非奇异矩阵

### 奇异矩阵：信息的一扇"单向门"

想象你在玩一个密室逃脱游戏。你走进了一扇门，门在你身后自动关上了。你回头一看，门上没有把手——你回不去了。这扇门就是"奇异的"：它允许信息通过（从左到右），但信息一旦过去，就无法原路返回。

**奇异矩阵（Singular Matrix）** 就是这么一扇门。它做的事情是：把输入向量变换成输出向量，但在这个过程中，某些信息被永久地丢失了，你无法从输出反推回输入。

数学上的定义很简单：**奇异矩阵就是行列式为零的方阵。**

$$\det(A) = 0 \iff A \text{ 是奇异矩阵}$$

但行列式为零是什么意思？让我们从多个角度来理解。

### 四个等价视角

对于方阵 $A$（$n \times n$），以下四个说法是完全等价的：

1. **行列式为零**：$\det(A) = 0$
2. **秩不满**：$\text{rank}(A) < n$
3. **不可逆**：不存在矩阵 $A^{-1}$ 使得 $A^{-1}A = I$
4. **存在非零向量被"压扁"为零**：存在 $\mathbf{x} \neq \mathbf{0}$ 使得 $A\mathbf{x} = \mathbf{0}$

这四个视角从不同角度描述了同一件事：**矩阵丢失了信息。**

让我们用 3D 到 2D 投影的类比来理解。想象你把一个三维物体投影到二维纸上。投影操作就是一个 $2 \times 3$ 的矩阵——它把 3D 坐标压缩成了 2D 坐标。但一旦压缩完成，你就无法从 2D 纸上的影子还原出 3D 物体的完整形状。深度信息丢了。

对于方阵来说，"奇异"意味着矩阵在自己的维度内做了类似的事情：它把 $n$ 维空间压缩到了低于 $n$ 维的子空间里。比如一个 $3 \times 3$ 的奇异矩阵，可能把三维空间压成了二维平面（秩为 2），甚至一条线（秩为 1），或者一个点（秩为 0）。

### 非奇异矩阵：信息的完美双行道

**非奇异矩阵（Non-singular Matrix）** 是奇异矩阵的反面。它是满秩的、可逆的、行列式不为零的。

用信息的角度来说：非奇异矩阵在变换过程中**没有丢失任何信息**。你总可以从输出完美还原回输入——反推的操作就是矩阵求逆 $A^{-1}$。

$$A^{-1}A = AA^{-1} = I$$

单位矩阵 $I$ 代表"什么都不做"的变换。所以 $A^{-1}A = I$ 意味着："先做 $A$，再做 $A^{-1}$，等于什么都没做"——变换被完美撤销了。

### 为什么神经网络初始化要避免权重矩阵奇异

这里有一个非常重要的实践问题。在训练神经网络时，权重矩阵 $W$ 是随机初始化的。如果初始化不慎导致某些权重矩阵接近奇异（甚至奇异），会发生什么？

**梯度消失或爆炸。** 当矩阵接近奇异时，它的某些奇异值接近零。反向传播时，梯度需要乘以 $W^T$（或相关的矩阵），接近零的奇异值会让某些方向的梯度信号被极度压缩，导致这些方向上的参数几乎不更新。

**表达能力受限。** 一个秩为 $r$ 的 $n \times n$ 矩阵，只能将 $n$ 维输入映射到 $r$ 维的子空间。如果权重矩阵的秩很低，那么无论输入多么丰富，输出都被限制在一个低维空间里。就像你有一个 1024 维的词嵌入，但如果中间的权重矩阵秩只有 64，那有效信息就被压缩到了 64 维——大量信息被浪费了。

因此，好的初始化方案（如 Xavier 初始化和 Kaiming 初始化）会确保：
- 权重矩阵的各奇异值大致均匀分布，不出现接近零的情况
- 前向传播时信号的方差保持稳定（不会逐层放大或缩小）
- 反向传播时梯度的大小保持稳定

Xavier 初始化的具体做法是让 $W$ 的每个元素从均匀分布 $U\left[-\frac{\sqrt{6}}{\sqrt{n_{in} + n_{out}}}, \frac{\sqrt{6}}{\sqrt{n_{in} + n_{out}}}\right]$ 中采样，其中 $n_{in}$ 和 $n_{out}$ 分别是输入和输出的维度。这个精心设计的范围确保了矩阵既不太接近奇异（奇异值不会太小），也不至于让输出爆炸（奇异值不会太大）。

### Python 演示：奇异 vs 非奇异

```python
import numpy as np

# 非奇异矩阵：满秩，可逆
A = np.array([[2, 1],
              [1, 3]])
print(f"A 的行列式: {np.linalg.det(A):.1f}")  # 5.0
print(f"A 的秩: {np.linalg.matrix_rank(A)}")   # 2
print(f"A 可逆? 是。A^(-1) =\n{np.linalg.inv(A)}")
print(f"验证 A^(-1) @ A =\n{np.linalg.inv(A) @ A}")  # 单位矩阵

# 奇异矩阵：行列式为零，不可逆
B = np.array([[1, 2],
              [2, 4]])  # 第二行是第一行的 2 倍
print(f"\nB 的行列式: {np.linalg.det(B):.1f}")  # 0.0
print(f"B 的秩: {np.linalg.matrix_rank(B)}")    # 1
try:
    np.linalg.inv(B)
except np.linalg.LinAlgError as e:
    print(f"B 可逆? 否。错误信息: {e}")

# 观察奇异矩阵如何"压扁"空间
x = np.array([2, -1])
print(f"\nB @ {x} = {B @ x}")  # [0, 0] — 非零向量被压成了零！

# 对比：非奇异矩阵不会把非零向量压成零
nonzero_found = False
for _ in range(1000):
    x_random = np.random.randn(2)
    if np.allclose(A @ x_random, 0):
        nonzero_found = True
        break
print(f"非奇异矩阵 A 把非零向量压成零? {nonzero_found}")  # False
```

注意那段关键的演示：奇异矩阵 $B$ 能把非零向量 $\begin{pmatrix} 2 \\ -1 \end{pmatrix}$ 压成零向量——这意味着这个方向上的信息被彻底消灭了。而非奇异矩阵 $A$ 不可能做到这一点：任何非零输入，经过 $A$ 变换后仍然是非零的。

## 4.4 秩与信息压缩

### 低秩：用更少参数存更多信息

让我们算一笔账。一个 $1000 \times 1000$ 的矩阵有一百万个元素。但如果这个矩阵的秩只有 10，它实际上只有 10 个"独立的"信息维度。你真的需要存一百万个数吗？

不需要。根据 SVD 分解 $A = U\Sigma V^T$，一个 $m \times n$、秩为 $r$ 的矩阵可以只用三个小矩阵来存储：

$$A \approx U_r \Sigma_r V_r^T$$

其中 $U_r$ 是 $m \times r$，$\Sigma_r$ 是 $r \times r$（对角矩阵），$V_r$ 是 $n \times r$。总参数量为 $mr + r + nr = r(m + n + 1)$。当 $r$ 远小于 $\min(m, n)$ 时，这是一个巨大的压缩。

对于 $1000 \times 1000$、秩为 10 的矩阵：
- 原始存储：$1000 \times 1000 = 1{,}000{,}000$ 个数
- 低秩存储：$10 \times (1000 + 1 + 1000) = 20{,}010$ 个数

**压缩比 50:1。** 50 倍的存储节省。

### JPEG 压缩的数学本质

你手机里的每张 JPEG 照片，背后都在用低秩近似的思想。

一张 $512 \times 512$ 的灰度图可以看作一个 $512 \times 512$ 的矩阵，每个元素是一个像素的灰度值。自然图像有一个重要特点：相邻像素高度相关——天空是渐变的蓝色，墙壁是均匀的白色。这意味着图像矩阵的奇异值通常**衰减很快**：前几个奇异值携带了绝大部分信息，后面的奇异值几乎可以忽略。

JPEG 压缩（更准确地说是其核心的 DCT 变换）做的事情就是：把图像分解成一组"基图像"的加权和，然后丢掉权重小的那些。这本质上就是保留前 $k$ 个奇异值，截断后面的：

$$A \approx \sum_{i=1}^{k} \sigma_i \mathbf{u}_i \mathbf{v}_i^T$$

其中 $k < r$。你丢掉了一些细节（那些权重 $\sigma_i$ 很小的层），但保留了图像的主体结构。$k$ 越小，压缩越狠，图像越模糊；$k$ 越大，图像越清晰，但压缩效果越差。

### 秩与"自由度"

秩和自由度（degrees of freedom）是紧密相关的概念。一个 $m \times n$、秩为 $r$ 的矩阵有 $r(m + n - r)$ 个自由度——这是完全确定这个矩阵所需的最少参数数量。

直觉上，自由度 = 你能自由选择的参数个数。满秩的 $n \times n$ 矩阵有 $n^2$ 个自由度（所有元素都可以自由选择），而秩为 1 的 $n \times n$ 矩阵只有 $2n - 1$ 个自由度（选择 $\mathbf{u}$ 和 $\mathbf{v}$ 就够了，但 $\mathbf{u}\mathbf{v}^T = (c\mathbf{u})(\frac{1}{c}\mathbf{v}^T)$，所以整体缩放不改变矩阵，要减掉 1）。

在机器学习中，"自由度"和"模型容量"直接相关。自由度太高的模型容易过拟合（记住噪声而非规律），自由度太低的模型容易欠拟合（连规律都学不到）。低秩约束，就是控制模型自由度的一种方式。

### 预演 LoRA：低秩微调大模型

现在让我们把所有概念串起来，看看线性代数如何直接催生了 LLM 领域最重要的技术之一：**LoRA（Low-Rank Adaptation）**。

微调一个大语言模型时，你需要更新它的权重矩阵。假设有一个 $4096 \times 4096$ 的权重矩阵 $W_0$（约 1600 万参数），全量微调需要更新所有 1600 万个参数——这在计算和存储上都非常昂贵。

LoRA 的核心观察是：**微调过程中，权重的变化量 $\Delta W$ 通常是低秩的。** 也就是说，虽然 $W_0$ 本身可能是满秩的，但 $W_{\text{fine-tuned}} - W_0 = \Delta W$ 的秩很低——微调只需要在少数几个方向上调整权重。

既然 $\Delta W$ 的秩为 $r$，根据秩一分解的思想，它可以写成：

$$\Delta W = BA$$

其中 $B$ 是 $4096 \times r$，$A$ 是 $r \times 4096$。当 $r = 8$ 时：

- 全量微调参数：$4096 \times 4096 = 16{,}777{,}216$
- LoRA 微调参数：$4096 \times 8 + 8 \times 4096 = 65{,}536$

**压缩比 256:1。** 只需调整原参数量的 0.4%，就能达到接近全量微调的效果。

这个惊人的效率，完全建立在"秩"这个概念之上。$\Delta W$ 的低秩性不是巧合——它反映了微调的本质：预训练模型已经学到了通用的语言知识，微调只是在少数几个"方向"上做定向调整，而不是推倒重来。

在 Transformer 的具体实现中，LoRA 通常应用在注意力层的 $W^Q$ 和 $W^V$ 投影矩阵上。前向传播变为：

$$Q = X(W^Q + B^Q A^Q)$$

其中 $B^Q A^Q = \Delta W^Q$ 就是低秩的权重更新。训练时 $W^Q$ 冻结，只更新 $B^Q$ 和 $A^Q$。推理时可以把 $\Delta W$ 预先合并回 $W^Q$，不增加推理延迟。

### Python 演示：低秩近似与信息压缩

```python
import numpy as np

# 构造一个"自然图像式"的矩阵：信号 + 噪声
np.random.seed(42)
m, n = 100, 100

# 创建一个低秩信号（秩为 5）
U_signal = np.random.randn(m, 5)
V_signal = np.random.randn(5, n)
signal = U_signal @ V_signal

# 加入少量噪声
noise = 0.1 * np.random.randn(m, n)
A = signal + noise

print(f"原始矩阵秩: {np.linalg.matrix_rank(A)}")  # 接近 100（因为噪声）

# SVD 分解
U, S, Vt = np.linalg.svd(A)
print(f"前 10 个奇异值: {np.round(S[:10], 1)}")

# 用前 5 个奇异值近似（因为真正的信号秩为 5）
k = 5
A_approx = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]

# 比较误差
relative_error = np.linalg.norm(A - A_approx) / np.linalg.norm(A)
print(f"\n用秩 {k} 近似的相对误差: {relative_error:.4f}")

# 存储对比
original_params = m * n
compressed_params = k * (m + n + 1)
print(f"原始参数量: {original_params}")
print(f"压缩后参数量: {compressed_params}")
print(f"压缩比: {original_params / compressed_params:.1f}:1")

# 观察奇异值的衰减
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(12, 5))

axes[0].semilogy(range(1, 21), S[:20], 'bo-')
axes[0].axvline(x=5, color='r', linestyle='--', label='真实秩 k=5')
axes[0].set_xlabel('奇异值编号')
axes[0].set_ylabel('奇异值 (log scale)')
axes[0].set_title('奇异值衰减')
axes[0].legend()
axes[0].grid(True, alpha=0.3)

# 不同 k 值下的近似误差
errors = []
ks = range(1, 30)
for ki in ks:
    A_k = U[:, :ki] @ np.diag(S[:ki]) @ Vt[:ki, :]
    errors.append(np.linalg.norm(A - A_k) / np.linalg.norm(A))

axes[1].plot(ks, errors, 'ro-')
axes[1].axvline(x=5, color='r', linestyle='--', label='真实秩 k=5')
axes[1].set_xlabel('近似使用的秩 k')
axes[1].set_ylabel('相对误差')
axes[1].set_title('低秩近似的误差 vs 秩')
axes[1].legend()
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig('low_rank_approx.png', dpi=150)
plt.show()
```

运行这段代码，你会看到奇异值在前 5 个之后急剧下降——这就是低秩结构的证据。右图显示，当 $k$ 达到 5 之后，继续增加 $k$ 对误差的改善微乎其微，因为那些额外的分量主要在拟合噪声。

### 在 Transformer 中的具体体现

让我们把视角拉回到 Transformer 论文公式级别。在原始 Transformer 的注意力机制中：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中 $Q = XW^Q$，$K = XW^K$，$V = XW^V$。对于 GPT-3 的某些配置，$W^Q$ 是一个 $12288 \times 12288$ 的矩阵。使用 LoRA 微调时：

$$Q = X(W^Q + B^Q A^Q)$$

其中 $B^Q$ 是 $12288 \times r$，$A^Q$ 是 $r \times 12288$。论文中作者发现即使 $r = 4$ 或 $r = 8$（极低的秩），就能在许多任务上达到与全量微调相当的效果。

**这个实验结果告诉我们一个深刻的事实：语言模型在微调时所需的"自由度"非常低。** 预训练阶段学到的知识已经足够丰富，微调只是在少数几个维度上做精细调整。秩，精确地量化了这个"少数"到底是多少。

## 小结

这一章我们从"信息"的角度重新审视了矩阵的几个核心属性：

1. **秩 = 独立信息量**。它告诉你矩阵里有多少列（或行）是在说不同的事情。秩越小，冗余越多。
2. **秩一矩阵是矩阵的"原子"**。任何矩阵都可以拆成若干秩一矩阵的和——这就是 SVD 的直觉基础。每一个秩一矩阵只携带一个"方向"的信息。
3. **奇异矩阵 = 信息的单向门**。行列式为零、秩不满、不可逆——四种等价的说法，描述的都是同一件事：矩阵在变换过程中不可逆地丢失了信息。
4. **低秩 = 压缩的机会**。如果一个矩阵的有效秩很低，就能用远少于原始参数量的数据来存储和传输它。JPEG 压缩是图像领域的经典应用，LoRA 是大模型微调领域的革命性应用。

下一章，我们将进入特征值与特征向量的世界——那些在线性变换中"坚守方向"的特殊向量，以及它们如何帮助我们理解 Transformer 的训练动态和注意力机制的本质。
